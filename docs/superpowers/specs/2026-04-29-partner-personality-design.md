# 伴侣人格测试小程序 - 完整设计文档

**项目名称：** 伴侣人格测试  
**版本：** 1.0  
**日期：** 2026-04-29  
**设计理念：** 场景化实用 + 适度搞怪

---

## 1. 产品概述

### 1.1 产品定位
一款年轻化、搞怪的人格评估小程序。用户通过 30 道题测评自己的伴侣/暗恋对象，系统根据加权计分生成独特的人格标签和详细解读。

### 1.2 目标用户
- 18-35 岁的恋爱中或有暗恋的年轻人
- 想要更了解伴侣/暗恋对象的人格特征

### 1.3 核心价值
- 帮助用户更客观地了解伴侣的人格特质
- 提供实用的相处建议
- 娱乐性强，易于分享

## 2. 核心机制

### 2.1 四维度人格系统

**维度 1：恋爱态度（Romance Attitude）**
- 衡量用户评估的伴侣在感情中的态度类型
- 4 个结果选项：
  1. **专一深情**：全心投入，不容许背叛，对感情认真负责
  2. **温柔暖男/暖女**：温和、贴心、会主动关心，情感表达温柔
  3. **浪漫花心**：热情洋溢、容易心动、喜欢新鲜感
  4. **理性冷静**：感情中相对理性，不过分依赖，看重实际

---

**维度 2：性格特质（Personality Traits）**
- 衡量伴侣的核心性格倾向
- 4 个结果选项：
  1. **甜蜜可爱**：温柔体贴、讨人喜欢、有点娇软
  2. **独立自信**：主见强、自我驱动、不依附他人
  3. **幽默风趣**：爱搞笑、活泼开朗、能调节气氛
  4. **沉静深邃**：内敛、思考深、神秘有魅力

---

**维度 3：相处方式（Interaction Style）**
- 衡量伴侣与你相处的倾向
- 4 个结果选项：
  1. **粘人缠绵**：喜欢贴近你、经常主动找你、很依赖你的陪伴
  2. **陪伴贴心**：关心你但不粘人、懂得给空间、细心周到
  3. **自在独立**：尊重双方空间、不过分依赖、各有各的生活
  4. **有点冷淡**：相处时有距离感、不太主动、需要你去靠近

---

**维度 4：沟通风格（Communication Style）**
- 衡量伴侣的表达和沟通特点
- 4 个结果选项：
  1. **温言细语**：说话温柔、很少直接批评、善于倾听
  2. **直言快语**：有话直说、不绕弯子、坦白率高
  3. **幽默逗趣**：喜欢用笑话缓解、爱开玩笑、能化解尴尬
  4. **沉默寡言**：不爱说话、需要时间才开口、内敛沉默

## 3. 题库设计

### 3.1 题库结构

- **共同题：** 20 道（所有用户都要答）
  - 维度 1（恋爱态度）：5 道题
  - 维度 2（性格特质）：5 道题
  - 维度 3（相处方式）：5 道题
  - 维度 4（沟通风格）：5 道题

- **男性专属题：** 10 道（选择"测他"时出现）
- **女性专属题：** 10 道（选择"测她"时出现）

- **总题数：** 30 道（共同 20 + 专属 10）

---

## 4. 计分算法

### 4.1 加权计分方法

**基本规则：**
1. 用户答题时，每选择一个非"不知道"选项，该选项对应的维度获得相应权重分
2. 每个维度计算所有获得的权重分总和
3. 维度得分最高的选项就是该维度的结果

**示例计算：**

假设用户回答了 30 道题，其中恋爱态度维度的得分如下：
- 专一深情：5 + 3 + 4 = 12 分
- 温柔暖男/暖女：2 + 3 = 5 分
- 浪漫花心：0 分
- 理性冷静：2 分

**结果：恋爱态度 = 专一深情**

### 4.2 维度有效性检查

如果某个维度的有效答题数少于 3 题（比如用户选了太多"不知道"），则该维度不可信，需要提示用户重新测试。

## 7. 技术规范（初步）

### 7.1 数据结构

**题目对象：**
```
{
  id: number,
  gender: 'both' | 'male' | 'female',
  category: 'attitude' | 'traits' | 'interaction' | 'communication',
  question: string,
  emoji: string,
  options: [
    { text: string, weights: { attitude?: number, traits?: number, interaction?: number, communication?: number } },
    ...
  ]
}
```

**用户答题结果对象：**
```
{
  userId: string,
  gender: 'male' | 'female',
  answers: { [questionId]: number }, // option index
  scores: { attitude: { [option]: number }, ... },
  result: { attitude: string, traits: string, interaction: string, communication: string },
  label: string,
  timestamp: date
}
```

### 7.2 计算流程

1. 收集用户答题数据
2. 按维度统计权重分数
3. 检查维度有效性（有效答题数 >= 3）
4. 为每个维度选择得分最高的选项
5. 组合生成最终标签
6. 查表返回对应的解读文案
